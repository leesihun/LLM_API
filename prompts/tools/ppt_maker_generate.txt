## Marp Syntax Rules

### Front Matter (Global Directives)
Always start with front matter:
---
marp: true
theme: {theme}
paginate: {paginate}
{footer_directive}
{header_directive}
---

### Slide Separators
- Use `---` on a new line to separate slides
- Each `---` creates a new slide

### Slide Content
- Use standard Markdown: `# Heading`, `## Subheading`, `**bold**`, `*italic*`
- Lists: `- Bullet` or `1. Numbered`
- Images: `![](image-url)` or `![bg](image-url)` for background
- Code blocks: ```language ... ```

### Per-Slide Directives (Optional)
Use HTML comments for per-slide settings:
- Background color: `<!-- _backgroundColor: #123456 -->`
- Text color: `<!-- _color: white -->`
- Custom class: `<!-- _class: lead -->`

### Special Features
- Auto-fit text: `# <!-- fit --> Large Title`
- Math: `$inline$` or `$$block$$`
- Two columns: Use HTML tables or `<div style="display: grid; grid-template-columns: 1fr 1fr;">`

## Output Requirements

1. **Start with front matter** (mandatory)
2. **Title slide first** with presentation title and subtitle/author if applicable
3. **Content slides** with clear structure:
   - Use `#` for slide titles
   - Use `##` for section headings within slides
   - Use bullet points for key information
   - Keep text concise (max 5-7 bullets per slide)
   - Use visual hierarchy consistently
4. **Maximum {max_slides} slides** (including title slide)
5. **Professional formatting**:
   - Consistent structure across slides
   - Clear visual hierarchy
   - Appropriate spacing
   - Professional language
   - No grammatical errors

## Presentation Design Best Practices

- **Title Slide**: Include main title and subtitle/author/date
- **Content Slides**: One main idea per slide
- **Text Amount**: Less is more - avoid walls of text
- **Lists**: Use bullets for unordered items, numbers for steps/sequences
- **Conclusion Slide**: Summarize key takeaways
- **Visual Balance**: Don't overcrowd slides
